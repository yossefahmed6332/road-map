The core idea

Backend trio

One unit, three phases

Yossef, Norhan, and Rafiq always move together. You design at the same time, code at the same time, and debug at the same time. Being in the same space means a blocker gets resolved in 30 seconds — not after a 20-minute call.

Frontend duo

Fully independent track

Asmaa and Ahmed never wait for backend. They build against a mock data file that has the exact same shape as the real API. When backend is ready, Ahmed swaps one import line. No rewrites. No surprises.

Design together
Architecture session +
All 3 sit together. Draw the DB schema, define every API route (method + URL + request body + response shape), and assign modules. This is your single design session — the output is a committed API_CONTRACT.md.
Yossef → DB schema, models, seed data
Norhan → core feature routes and controllers
Rafiq → server entry, auth routes, JWT middleware
Running in parallel
Foundation setup +
Don't wait for backend to finish designing. Start immediately.
Asmaa builds full HTML structure across all pages
Ahmed sets up Vite + React Router + folder structure
As soon as the contract is pushed, Ahmed creates mockData.js matching the exact response shapes
0:45 3h 15min
Code together
Parallel module coding +
"Together" means at the same time and in the same space — each person codes their own module, but blockers get resolved in seconds because you're right there.
No one touches another person's files. Strict ownership.
Stuck for 10+ min? Say it out loud immediately — don't silently struggle for 40 minutes.
Push to your branch every 30–45 min. Small, frequent commits.
Running in parallel
Full React build +
Ahmed builds all React components wired to mock data. Asmaa polishes CSS across all pages, loading states, and empty states. Frontend should be feature-complete by hour 4 — ahead of backend on purpose.
Mock data has the exact same shape as the API contract.
Swapping to real API = one import line, not a component rewrite.
4:00 1 hour
Debug together
Merge and test all routes +
All 3 merge to dev together. Run the full server and test every contract endpoint with Thunder Client or Postman. Fix bugs together — three sets of eyes on the same screen.
One person tests, two read the response and spot the issue.
If a route is broken, the module owner fixes it while the other two watch and help.
Don't split off to debug alone. The whole point is three eyes on every bug.
Buffer time
Edge cases + polish +
Frontend is done or near-done. Use this hour to handle edge cases: empty states, error messages, responsive layout, loading skeletons. Being done early is the goal — it gives integration room to breathe.
5:00 1 hour
Everyone together — Integration sprint
Connect frontend to live backend +
Ahmed swaps mock data for real API calls. Everyone walks through each user flow end-to-end. Backend devs stay on standby — if a route returns wrong data, the module owner fixes it immediately while others keep testing other flows.
Walk the demo click path first. Fix that flow before anything else.
P1 = demo-breaking. P2 = annoying. P3 = cosmetic. Only fix P1s.
One person owns CORS and env config — don't let three people debug the same config file.
6:00 1 hour
Everyone — Demo polish
Freeze, seed, rehearse +
Seed the DB with realistic demo data — nothing kills a demo like empty screens. Assign one calm presenter (not the one who built the feature). Rehearse the exact click path twice.
No new features after 6:30. Freeze the code. Stability over scope.
Record a 2-minute screen recording as a backup if the live demo crashes.
04

Communication rules

No meetings. All async. Only GitHub.

Rule 01
The 10-minute rule
If you're stuck for more than 10 minutes, say it out loud (if together) or drop a GitHub comment on your own PR tagging the relevant person. Don't call a meeting.
Rule 02
Small PRs only
Every PR is focused and small. No one pushes a 400-line commit. Small PRs get reviewed in 3 minutes. Giant PRs cause merge conflicts and arguments.
Rule 03
The contract is law
If a backend route needs to change shape, update API_CONTRACT.md AND immediately notify Ahmed. He updates his mock data to match. No surprises during integration.
Rule 04
One merge owner
Yossef handles all merges to main. Everyone else merges to dev only. This prevents the classic "who broke main?" situation at hour 5.
